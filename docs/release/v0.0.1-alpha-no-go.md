# v0.0.1 Alpha GitHub Release No-Go Report

Date: 2026-06-23

Status: Superseded on 2026-06-24 by the reviewed source-archive release. See
[v0.0.1 alpha release report](./v0.0.1-alpha-release.md) and the published
[GitHub Release](https://github.com/kanohorizonia/kano-git-master-skill/releases/tag/v0.0.1).

This file remains as historical evidence for the earlier no-go decision.

## Decision

Do not publish GitHub Release `v0.0.1` yet.

The release tag exists and GitHub permissions are available, but the release
artifact preflight did not produce a validated archive, manifest, and checksum
from the `v0.0.1` tag. Publishing now would imply release assets are ready when
they are not.

## Verified

- `git cat-file -t v0.0.1` reports `tag`, so `v0.0.1` is annotated.
- `git show --no-patch --format=fuller v0.0.1` shows tag message
  `kano-git-master-skill 0.0.1 alpha`.
- The tag points to commit `2f61fe6dfe9eb28b89f8eff587f4a0084e69eaac`.
- `git ls-remote --tags origin v0.0.1` reports remote tag object
  `0491f8b275c180f9919f667547ae78281e84b16b`.
- `gh auth status` is authenticated to `github.com` as `dorgonman` with
  `repo` and `workflow` scopes.
- `gh release view v0.0.1 --repo kanohorizonia/kano-git-master-skill` reports
  `release not found`, so no GitHub Release was present before this preflight.

## Release Action Taken

No GitHub Release was created.

No package-manager, binary, or downloadable artifact claim should be made for
`v0.0.1` until the tag can produce a validated release archive or a maintainer
chooses a replacement release strategy.

## Required Next Step

Pick one release path:

- fix release export/build on `main` and publish a new `v0.0.2` alpha tag; or
- if maintainers explicitly approve replacing the public tag, rebuild the
  `v0.0.1` release candidate from a corrected commit and force-update the tag.

The safer default is a new `v0.0.2` alpha because `v0.0.1` is already published
as a remote tag.
